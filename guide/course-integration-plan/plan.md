# 课程整合实施计划

## 🔗 知识依赖图谱

```text
第一周：AI Agents 驱动快速实现
    ├─ Markdown/GitHub 基础
    ├─ AI Agent 对话技巧
    ├─ 与 Analyst 对话 → project-brief.md
    ├─ 与 PM 对话 → prd.md
    ├─ 与 UX Expert 对话 → front-end-spec.md (初稿)
    ├─ 与 Architect 对话 → architecture.md
    ├─ 准备素材（图片 + 素材描述 assets/README.me 生成 data.js）
    └─ 与 Dev Agent 对话 → 生成网站初稿代码
         ↓
         【成果：可运行的作品集网站】
         ↓
第二周：逆向设计系统（从网站反推规范）
    ├─ 分析已生成网站的视觉元素
    ├─ Figma 操作与组件化思维
    ├─ Design Tokens 原理
    ├─ 在 Figma 中重建设计系统
    ├─ 更新 design-tokens.md（规范化）
    └─ 更新 front-end-spec.md（补充设计系统）
         ↓
         【成果：规范的设计生态】
         ↓
第三周：基于设计系统的迭代开发
    ├─ 应用设计系统优化代码
    ├─ HTML/CSS/JS 深入理解
    ├─ 响应式设计与无障碍性
    ├─ 性能优化与部署
    └─ 产品进化思维
         ↓
         【成果：可维护可扩展的专业作品集】
```

### 🎯 教学策略说明

- **先快后慢**：第一周快速看到成果（激发兴趣），后续逐步深入专业化
- **逆向工程**：从已有网站反推设计系统（比从零学设计更直观）
- **Agent 协作**：学生通过与不同角色的 AI 对话，理解软件开发的完整流程

## 💡 差异化价值主张

相比传统Web开发课程，本课程的独特价值：

1. **AI First**：不是学完再用AI，而是从第一天就与AI协作
2. **系统思维**：不是教代码片段，而是教设计系统和工作流
3. **认知升级**：从"工具使用者"培养为"系统架构者"
4. **长期导向**：强调可维护性和持续进化，而非一次性交付

---

## 📌 关键成功因素

1. **严格控制每节课的时长**：不超过90分钟，保护注意力
2. **强制性课间作业**：确保知识巩固和成果积累
3. **同伴反馈机制**：每节课都有学员展示环节
4. **教师示范质量**：避免技术故障，提前测试所有流程
5. **学员多样性管理**：为不同基础的学员提供差异化支持

## 📅 实施建议

### 推荐课程间隔与作业目标

#### 第一周 → 第二周：间隔 3-5 天

##### 核心目标：通过 AI Agents 对话生成网站初稿

学生需完成：

1. **与 Analyst Agent 对话** → 生成 `project-brief.md`
2. **与 PM Agent 对话** → 生成 `prd.md`（产品需求文档）
3. **与 UX Expert Agent 对话** → 生成 `front-end-spec.md`（前端规范）
4. **与 Architect Agent 对话** → 生成 `architecture.md`（架构文档）
5. **准备素材** → 整理 5-10 张作品图片 + 编写 `素材描述 assets/README.me 生成 data.js`（素材描述数据）
6. **与 Dev Agent 对话** → 基于以上文档，直接生成网站初稿代码

**重要说明**：此阶段的 `front-end-spec.md` 和 `design-tokens`（如有）是初步版本，目的是让学生快速看到成果，建立信心。

---

#### 第二周 → 第三周：间隔 5-7 天

##### 核心目标：逆向工程 - 从网站反推设计系统

学生需完成：

1. **在 Figma 中重建设计**：
   - 分析已生成网站的视觉元素
   - 在 Figma 中创建规范的设计系统（颜色、字体、间距、组件等）
2. **更新设计规范文档**：
   - 根据 Figma 设计，更新 `design-tokens.md`（更专业和系统化）
   - 更新 `front-end-spec.md`（补充设计系统规范）
3. **建立设计生态**：
   - 理解"设计驱动开发"的价值
   - 为未来迭代建立可维护的设计基础

##### 教学价值

这种"逆向"方法让学生：

- 先体验"快速交付"的成就感
- 再理解"设计系统"的必要性（解决维护和扩展问题）
- 培养"先实现后优化"的实战思维

### 混合教学模式

- 前70%时间：教师演示 + 同步实践
- 后30%时间：学员独立操作 + 教师巡回指导

### 在线vs线下

- **线下优势**：实时指导、技术问题快速解决
- **在线优势**：录屏回放、学员可按自己节奏
- **建议**：线上线下混合，关键演示线下完成
