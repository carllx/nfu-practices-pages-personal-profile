### 第二部分：AI智能体（Agent）的革命性概念（20分钟）

#### 1. 痛点引入（5分钟）
- **为什么普通AI输出不稳定？**
- **对比演示**：同样的需求，通用AI vs Agent的输出差异

#### 2. Agent = 角色化的专家 Prompt（10分钟）
- **核心概念**：不是聊天机器人，是你的虚拟团队成员
- **agents文件夹导览**（打开几个.txt文件展示）：
  - `analyst.txt`：分析师Mary，负责需求挖掘和项目简报
  - `pm.txt`：产品经理，负责PRD文档
  - `ux-expert.txt`：UX专家，负责前端规范
  - `architect.txt`：架构师，负责技术架构
  - `dev.txt`：前端开发者，负责写代码

#### 3. 关键约束：技术栈锁定（5分钟）
- ⚠️ **必须强调**：限制为纯 HTML + CSS + JavaScript
- **原因**：React/Vue对初学者是黑盒，无法理解和调试
- **强制话术模板**：每次与AI对话必须包含
  ```
  技术限制：严格使用 HTML, CSS, JavaScript，
  不使用 React、Vue 等框架。
  ```

#### 4. 模型选择策略（基于实践经验）
- **讨论阶段**：DeepSeek（深度思考模式）/ GPT / Gemini
- **代码生成**：智谱（在VS Code Agent模式中）
- **原因**：智谱适合执行，不适合头脑风暴

## 📋 课间作业（3-5天完成）⭐

这是整个课程最关键的产出期！学员需要通过与不同AI Agent对话，生成完整的网站初稿。

### 核心原则
1. **顺序执行**：按照 Analyst → PM → UX Expert → Architect → Dev 的顺序
2. **文档传递**：每一步的输出是下一步的输入
3. **反复强调**：在每次对话中都要提及"纯 HTML/CSS/JavaScript"限制
4. **利用澄清机制**：在对话中加入"如果有不确定的地方请主动询问我"
5. **不追求完美**：这是初稿，目标是快速看到成果

---

### 任务1：与 Analyst Agent 对话 → 生成 `docs/brief.md`

**核心要点**：

用户需要提供的基本需求：
- "我想为艺术家 carllllllllx 创建一个个人作品集网站"
- **明确页面需求**：需要几个页面？每个页面的用途？（如：首页、关于、作品展示等）
- 明确风格要求（如"1930年代橡皮管动画风格"）
- 指定技术限制（如"纯 HTML/CSS/JavaScript"）
- "如果有不确定的地方请你主动询问我" 利用这种机制及时澄清模糊需求

![Analyst Agent 对话示例](https://i.imgur.com/otaaPN5.webp)

**操作步骤**：

1. **准备工作**
   - 打开 DeepSeek / ChatGPT / Gemini（推荐使用深度思考模式）
   - 在对话框中上传或拖进 `agents/analyst.txt` 文件

2. **初始对话模板**
   ```markdown
   你好，Mary（分析师的名字）。
   
   我想为[你的身份，如：插画艺术家/摄影师/设计师]创建一个个人作品集网站。
   
   风格要求：[描述你喜欢的风格，如：极简主义/复古/现代科技感]
   
   页面需求：我需要 4 个页面
   - 首页（Home）：展示个人介绍和精选作品 + 代表作展示 + 导航
   - 关于页面（About）：艺术家简介和经历、创作理念、联系方式
   - 插画作品页（Illustration）：展示插画作品集, 12 张原创插画作品展示（按年份分类）
   - 产品设计页（Product）：展示产品设计作品集, 12 个衍生产品（与插画一一对应）

   技术限制：严格使用纯 HTML、CSS、JavaScript，
   不使用 React、Vue 等框架。
   
   如果有不确定的地方请你主动询问我。
   
   请帮我进行初步分析。
   ```

3. **多轮对话要点**
   - Analyst会主动提问（目标受众、作品数量、色彩偏好等）
   - 认真回答每个问题，提供具体信息
   - 可选择"头脑风暴"进一步探索创意方向
   - ⚠️ 每次提到技术需求时，都要重申"纯 HTML/CSS/JavaScript"

4. **最终生成指令**
   ```
   现在让我们创建项目简报文档 brief.md
   ```

5. **验收标准**
   - brief.md 包含：
     - ✅ 项目概述
     - ✅ 目标用户
     - ✅ **页面结构说明**（4个页面及其用途）
     - ✅ 风格定位
     - ✅ 技术约束
     - ✅ 设计方向

---

### 任务2：与 PM Agent 对话 → 生成 `docs/prd.md`

**核心要点**：

上传项目简报文件：
- brief.md（项目简报）

使用话术：
> "我们现在需要生成 `prd.md`，我提供了一个和分析师讨论的结果就是这里的 `brief.md`"

**关键点**：
- 明确告诉PM需要 **4个页面** 的具体功能设计
- 每个页面应该有清晰的功能列表和交互说明

⚠️ **重要提示**：当 PM 提出生成 Epic 列表时可以终止，强行生成 prd.md

![PM Agent 对话示例](https://i.imgur.com/tH50HZU.webp)

**操作步骤**：

1. **准备工作**
   - 上传或引用 `agents/pm.txt`
   - 准备好上一步生成的 `brief.md`

2. **初始对话模板**
   ```markdown
   你好，我们现在需要生成产品需求文档 `prd.md`。
   
   我已经和分析师讨论过项目，生成了 `brief.md`（见附件）。
   
   请基于这份简报，帮我制定详细的产品需求文档。
   
   明确需求：
   - 网站需要 4 个页面：首页、关于页、插画作品页、产品设计页
   - 请为每个页面定义具体功能和交互逻辑
   
   技术限制：纯 HTML、CSS、JavaScript。
   ```

3. **交互要点**
   - 当PM提出生成"Epic列表"（史诗列表）时，可以选择：
     - 继续深入讨论Epic
     - 或直接要求："现在直接生成完整的 prd.md"
   - PM可能会询问具体功能优先级，根据你的需求回答

4. **验收标准**
   - prd.md 包含：
     - ✅ **4个页面的详细功能说明**（首页、关于、插画页、产品页）
     - ✅ 用户故事
     - ✅ 页面间导航逻辑
     - ✅ 交互规范

### 任务3：与 UX Expert Agent 对话 → 生成 `docs/front-end-spec.md`（初稿）

**核心要点**：

上传项目简报文件：
- brief.md（项目简报）
- prd.md（产品需求文档）

使用话术：
> "我们制定设计规范的文档 front-end-spec.md"

明确表达具体需求：
- "我们的设计仅支持电脑端浏览器"
- Agent 会简化部分的设计策略

![UX Expert Agent 对话示例](https://i.imgur.com/EdDr9aH.webp)

**操作步骤**：

1. **准备工作**
   - 上传或引用 `agents/ux-expert.txt`
   - 准备 `brief.md` 和 `prd.md`

2. **初始对话模板**
   ```
   你好，我已经完成了项目简报（brief.md）和产品需求文档（prd.md）。
   
   请基于这些文档，帮我制定前端设计规范(front-end-spec.md)。
   
   补充说明：
   - 我们的设计仅支持电脑端浏览器（桌面端）
   - 技术限制：纯 HTML、CSS、JavaScript
   ```

3. **交互要点**
   - 明确告知是否需要移动端适配（初学者建议先不做）
   - 如果Agent建议复杂的设计工具，可以要求简化
   - 重点关注：颜色系统、字体、间距、组件规范

4. **验收标准**
   - front-end-spec.md 包含：Design Tokens（设计令牌颜色, 包括字体、间距）、组件规范、布局系统

⚠️ **重要提示**：这是初稿版本，第二周会在Figma中规范化和完善

### 任务4：准备素材 → 创建 `assets/README.md`

**为什么先准备素材？**  
先准备好实际素材，让架构师能基于真实情况设计最简单的数据结构！

**核心理念：用人类语言描述驱动 AI 生成数据结构**  
你的 `assets/README.md` 不仅是素材清单，更是给 AI 的"设计说明书"。描述越具体、关系越清晰，AI 生成的数据结构就越精准。

---

#### 📚 **理解素材之间的关系**

以本项目为例（参考 `assets/data.js`）：
- **插画作品**（Illustration）：你的原创视觉创意

- **对应关系**：每个插画可以衍生出相关产品（通过 `id` 和共享的 `alt` 文本关联）

```
插画 #1: "书页间流淌的光" 
    ↓ 灵感转化
产品 #1: "阅读之光猫咪台灯"（¥268）
```

```markdown
## 数据结构提示（给 AI 架构师的指令）
```

#### 💡 **进阶技巧**

**描述的具体程度决定数据质量**：

| 模糊描述 ❌ | 具体描述 ✅ |
|------------|------------|
| "一个台灯产品" | "陶瓷底座的猫咪造型台灯，LED暖光，高20cm，售价¥268" |
| "灵感来自插画" | "延续插画中书与光的意象，通过猫咪守护者的形态传递陪伴感" |

### 任务6：与 Dev Agent 对话 → 生成网站初稿代码

**操作步骤**：

1. **准备工作**
   - 打开 VS Code
   - 打开你的项目文件夹
   - 确保所有文档都已生成（brief, prd, front-end-spec, architecture）

2. **使用智谱 AI Agent 模式**
   - 在 VS Code 中打开智谱 AI 插件
   - 切换到 "Agent" 模式（不是 "Ask a question"）
   - 打开 `agents/dev.txt` 文件

3. **引用文档并下达指令**
   - 点击输入框旁边的 `@` 符号
   - 依次引用文件：
     - @brief.md
     - @prd.md
     - @front-end-spec.md
     - @architecture.md
     - @assets/README.md（如果有）
   
   - 输入指令：
     ```
     基于以上所有文档，请帮我生成个人作品集网站的代码。
     
     请依次生成：
     1. index.html（首页）
     2. about.html（关于页面）
     3. illustration.html（插画作品页）
     4. product.html（产品设计页）
     5. style.css（样式文件）
     6. script.js（交互脚本）
     
     技术要求：纯 HTML、CSS、JavaScript，使用语义化标签。
     ```

4. **分步生成与测试**
   - Dev Agent会逐步生成代码
   - 每生成一个文件，就在浏览器中测试
   - 如果遇到问题，描述问题让Agent修正
   - **确保4个页面都生成完整**：首页、关于、插画页、产品页

5. **验收标准**
   - ✅ **4个HTML页面都已生成**（index.html, about.html, illustration.html, product.html）
   - ✅ 能在浏览器中正常打开每个页面
   - ✅ 页面间导航功能正常
   - ✅ 页面显示正常，无明显错误
   - ✅ 基本交互功能可用（导航、图片展示等）
